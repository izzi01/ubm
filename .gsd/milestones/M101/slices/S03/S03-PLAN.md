# S03: PRD Import Bridge

**Goal:** Parse BMAD PRD markdown files and import functional/non-functional requirements into the GSD database, making them available as tracked requirements.
**Demo:** User runs /gsd import _bmad-output/planning-artifacts/prd.md, system extracts functional requirements (FR1-FR6) and creates corresponding GSD requirements. Requirements appear in REQUIREMENTS.md.

## Must-Haves

- `parsePrd()` correctly extracts all FR1-FR16 and NFR1-NFR6 from the real PRD\n- `/gsd import _bmad-output/planning-artifacts/prd.md` inserts requirements into DB and renders REQUIREMENTS.md\n- Re-importing the same file is idempotent (no duplicates)\n- Unit tests pass for parser edge cases

## Tasks

- [x] **T01: Build PRD markdown parser and tests** `est:45m`
  Create a pure function `parsePrd(content: string)` that extracts functional and non-functional requirements from BMAD PRD markdown. The parser targets the specific format in `_bmad-output/planning-artifacts/prd.md`:

1. **FR entries** live under `## Functional Requirements` → `### <category>` subsections, formatted as `* **FR1:** description text`
2. **NFR entries** live under `## Non-Functional Requirements` → `### <category>` subsections, formatted as `* **NFR1 (name):** description text`

The parser should:
- Return `ParsedRequirement[]` where each item has `{ id, class, description, source }`
- Class is `"functional"` for FR* entries, `"non-functional"` for NFR* entries
- Source is the PRD file path (passed as parameter)
- Strip markdown bold markers from descriptions
- Handle the NFR parenthetical name pattern: `NFR1 (Fault Tolerance)` → id=`NFR1`
- Be resilient to minor format variations (extra spaces, trailing periods)
- Return an empty array for PRDs with no requirement sections

Write thorough unit tests covering: happy path with FR+NFR extraction, NFR with parenthetical names, PRD with no requirements sections, malformed lines (skipped gracefully), multiple categories under one section, and empty input.
  - Files: `src/import/prd-parser.ts`, `src/import/types.ts`
  - Verify: npx vitest run tests/import/prd-parser.test.ts

- [x] **T02: Wire /gsd import command with DB persistence and REQUIREMENTS.md render** `est:45m`
  Create the `/gsd import <file-path>` slash command that:

1. Reads the PRD file from disk (relative to cwd or absolute path)
2. Calls `parsePrd()` to extract requirements
3. Deduplicates against existing DB requirements by ID (skip if already exists)
4. Inserts new requirements via `engine.db.requirementInsert()`
5. Renders `.gsd/REQUIREMENTS.md` as a markdown table of all requirements

Implementation steps:

1. Create `src/import/requirements-renderer.ts` — pure function `renderRequirementsMarkdown(requirements: RequirementRow[]): string` that produces a markdown table with columns: ID | Class | Description | Status | Source
2. Create `src/import/gsd-import.ts` — command handler factory `createImportHandler(engine)` following the same pattern as `gsd-commands.ts`. Handler:
   - Validates file exists and is readable
   - Parses with `parsePrd(content, filePath)`
   - Reports how many requirements were found vs already imported
   - Inserts new ones with status="active"
   - Renders REQUIREMENTS.md to `.gsd/REQUIREMENTS.md`
   - Shows summary via `ctx.ui.notify()` and `ctx.ui.setWidget()`
3. Register the command in `src/extension/index.ts` as `gsd import`
4. Export the import handler from `src/commands/index.ts`
5. Write integration test: create in-memory engine, parse the real PRD fixture, insert requirements, verify DB state and REQUIREMENTS.md output

Key constraints:
- Use the same `CommandHandler` type and factory pattern as gsd-commands.ts
- REQUIREMENTS.md must include a header comment: `<!-- Auto-generated by /gsd import. Do not edit manually. -->`
- On re-import, only insert requirements not already in DB (check by ID)
- Error handling: file not found → notify error, parse returns empty → notify "no requirements found"
  - Files: `src/import/gsd-import.ts`, `src/import/requirements-renderer.ts`, `src/extension/index.ts`, `src/commands/index.ts`
  - Verify: npx vitest run tests/import/gsd-import.test.ts
