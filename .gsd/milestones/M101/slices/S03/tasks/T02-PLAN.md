# T02: Wire /gsd import command with DB persistence and REQUIREMENTS.md render

Create the `/gsd import <file-path>` slash command that:

1. Reads the PRD file from disk (relative to cwd or absolute path)
2. Calls `parsePrd()` to extract requirements
3. Deduplicates against existing DB requirements by ID (skip if already exists)
4. Inserts new requirements via `engine.db.requirementInsert()`
5. Renders `.gsd/REQUIREMENTS.md` as a markdown table of all requirements

Implementation steps:

1. Create `src/import/requirements-renderer.ts` — pure function `renderRequirementsMarkdown(requirements: RequirementRow[]): string` that produces a markdown table with columns: ID | Class | Description | Status | Source
2. Create `src/import/gsd-import.ts` — command handler factory `createImportHandler(engine)` following the same pattern as `gsd-commands.ts`. Handler:
   - Validates file exists and is readable
   - Parses with `parsePrd(content, filePath)`
   - Reports how many requirements were found vs already imported
   - Inserts new ones with status="active"
   - Renders REQUIREMENTS.md to `.gsd/REQUIREMENTS.md`
   - Shows summary via `ctx.ui.notify()` and `ctx.ui.setWidget()`
3. Register the command in `src/extension/index.ts` as `gsd import`
4. Export the import handler from `src/commands/index.ts`
5. Write integration test: create in-memory engine, parse the real PRD fixture, insert requirements, verify DB state and REQUIREMENTS.md output

Key constraints:
- Use the same `CommandHandler` type and factory pattern as gsd-commands.ts
- REQUIREMENTS.md must include a header comment: `<!-- Auto-generated by /gsd import. Do not edit manually. -->`
- On re-import, only insert requirements not already in DB (check by ID)
- Error handling: file not found → notify error, parse returns empty → notify "no requirements found"
